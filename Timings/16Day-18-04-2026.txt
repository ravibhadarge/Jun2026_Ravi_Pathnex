2 ways to deploy application --->
1. Standard Production flow using NGINX INGRESS 
2. Service MEsh flow using ISTIO

1. nGINX INGRESS 
When we deploy our application using k8s deploymet ,
then we expose our application internally using Cluster IP Service 
next we install NGINX INGRESS CONTROLLER inside Cluster , 
when we install nginx ingress controller , a cloud load balancer is automatically provisioned ,
then we map domain name to load balancer IP using DNS 
after that we create a Ingress resource to define routing rules 
Finally Request flow through the system in this order .

2 Service MEsh flow using ISTIO 
first we deploy our application using deployment , but here each pod get a sidecar proxy,
then we expose our application internally using Cluster IP Service 
nexxt we install ISTIO which set up the Service MEsh inside the cluster ,
Instead of NGINX INGRESS , we use ISTIO INGRESS GATEWAY
A cloud Load Balancer is created and attached to ISTIO Gateway 
then we map domain to Load Balancer using DNS 
INstead of Ingress resource ,  ISTIO uses Virtual Service for routing .
